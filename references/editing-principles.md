# Editing Principles

Use this file as the house style for every personal statement review in this workspace. If a single template conflicts with this file, follow this file.

## Priority Order

Fix issues in this order:

1. Logic and paragraph function
2. Wording and sentence clarity
3. Rhythm and concision
4. Stylistic polish

## Default House Style

Prefer writing that is:

- clear rather than decorative
- sincere rather than grand
- specific rather than abstract
- controlled rather than overconfident
- logically progressive rather than repetitive

## What a Strong Draft Usually Does

- The opening lands on one concrete reason the applicant cares about the field.
- Each paragraph has one main job.
- Experience paragraphs show a link from action to insight to future direction.
- The fit paragraph explains "why this program or role" with evidence, not flattery.
- The conclusion moves forward instead of restating the opening.

## Common Problems to Cut

- generic "ever since I was young" openings
- empty modifiers such as "profound," "deeply," or "extremely" when they add no information
- repeated claims of passion, curiosity, or impact
- long sentences that delay the main point
- program praise that is not tied to the applicant's goals
- paragraphs that both narrate experience and argue fit at the same time

## Feedback Tone

Feedback should sound like a real reviewer, not a rubric bot.

- Be direct but not harsh.
- Keep most section comments to 2-4 sentences.
- Use natural phrasing such as "This part has material, but the logic is still a little loose" or "The paragraph is doing too much at once."
- Avoid over-praising before giving a correction.

## Non-Negotiables

- Do not invent facts.
- Do not import sentences from templates too literally.
- If a fact seems missing, mark it clearly as `[To confirm: ...]`.
- If a contradiction changes meaning, flag it instead of silently repairing it.
